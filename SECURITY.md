# Security Policy

## Supported Versions

Security fixes are provided for the latest code on the default branch.

## Reporting a Vulnerability

If you discover a security issue, do not open a public issue.

Send a private report with:
- A clear description of the vulnerability
- Steps to reproduce
- Expected impact
- Any proof-of-concept details

Contact the repository owner directly on GitHub:
- `https://github.com/TOUTAKUN04`

## Disclosure Policy

- Reports are reviewed privately.
- Public disclosure should wait until a fix is available.
- Unauthorized testing against third-party systems is not permitted.

## Security Notes for This Project

- File editing is permission-gated per session.
- File access should be limited to trusted roots.
- Adapter/webhook integrations should only be enabled with valid credentials and strict network controls.
